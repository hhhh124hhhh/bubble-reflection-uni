# 贡献者行为准则

## 我们的承诺

为了营造一个开放和友好的环境，我们作为贡献者和维护者承诺，无论年龄、体型、残疾、种族、性别认同和表达、经验水平、教育程度、社会经济地位、国籍、个人形象、种族、宗教或性取向如何，参与我们项目和社区的每个人都能享受无骚扰的体验。

## 我们的标准

有助于创造积极环境的行为包括：

- 使用友好和包容的语言
- 尊重不同的观点和经验
- 优雅地接受建设性批评
- 关注对社区最有利的事情
- 对其他社区成员表示同理心

不可接受的行为包括：

- 使用性化的语言或图像，以及不受欢迎的性关注或性骚扰
- 恶意评论、侮辱/贬损评论，以及个人或政治攻击
- 公开或私下骚扰
- 未经明确许可，发布他人的私人信息，如物理或电子地址
- 在专业环境中可能被合理认为不当的其他行为

## 我们的责任

项目维护者有责任澄清可接受行为的标准，并期望对任何不可接受的行为采取适当和公平的纠正措施。

项目维护者有权利和责任删除、编辑或拒绝与本行为准则不符的评论、提交、代码、wiki 编辑、问题和其他贡献，或者暂时或永久禁止任何他们认为有不适当、威胁、冒犯或有害行为的贡献者。

## 适用范围

本行为准则适用于项目空间内，也适用于个人在项目空间外代表项目或其社区时的情形。代表项目或社区的示例包括使用官方项目电子邮件地址、通过官方社交媒体账户发布内容，或在在线或线下活动中担任指定代表。项目的代表可以由项目维护者进一步定义和澄清。

## 执行

可以通过 [project-email@example.com](mailto:project-email@example.com) 联系项目团队来报告辱骂、骚扰或其他不可接受的行为。所有投诉都将被审查和调查，并将产生被认为必要和适合情况的回应。项目团队有义务对事件报告者保密。具体执行政策的更多细节可能会单独发布。

不善意遵守或执行行为准则的项目维护者可能面临项目领导层其他成员确定的临时或永久后果。

## 社区指导原则

### 🤝 互相尊重

- 尊重不同的技术观点和经验水平
- 承认每个人都有不同的背景和视角
- 避免使用可能排斥某些群体的语言

### 💬 建设性沟通

- 提供具体、可操作的反馈
- 专注于代码和想法，而不是个人
- 在讨论中保持耐心和理解

### 🌱 包容性

- 欢迎新贡献者并帮助他们融入社区
- 解释技术概念时考虑不同经验水平
- 创造一个让每个人都感到受欢迎的环境

### 🎯 专注于目标

- 记住我们的共同目标是改进心迹泡泡项目
- 将讨论集中在技术问题和解决方案上
- 避免偏离主题的争论

### 📚 持续学习

- 保持开放的心态，愿意学习新事物
- 承认错误并从中学习
- 分享知识和经验来帮助他人成长

## 具体场景指导

### 代码审查

**应该做：**
- 提供具体、建设性的反馈
- 解释为什么建议某种方法
- 承认代码中的优点
- 提供学习资源或示例

**不应该做：**
- 使用贬损或嘲笑的语言
- 只说"这是错的"而不解释原因
- 批评作者而不是代码
- 忽视代码审查礼仪

### 问题讨论

**应该做：**
- 保持讨论专注于技术问题
- 尊重不同的意见和解决方案
- 提供证据支持你的观点
- 在达成共识后愿意妥协

**不应该做：**
- 进行人身攻击
- 坚持己见而不考虑其他观点
- 使用煽动性或攻击性语言
- 转移话题或偏离讨论

### 新手支持

**应该做：**
- 耐心回答基础问题
- 提供学习资源和链接
- 分享你自己的学习经历
- 鼓励继续学习和探索

**不应该做：**
- 因为问题基础而嘲笑或轻视
- 使用过于复杂的技术术语
- 拒绝解释基本概念
- 表现出不耐烦或厌烦

## 报告流程

如果您遇到或目睹了违反本行为准则的行为，请：

1. **立即报告**: 不要等待，尽快报告问题
2. **提供细节**: 包括时间、地点、涉及人员和具体行为
3. **选择渠道**: 
   - 私信项目维护者
   - 发送邮件到：conduct@your-domain.com
   - 使用 GitHub 的报告功能

### 报告后会怎样

1. **确认收到**: 我们会在 24 小时内确认收到您的报告
2. **调查**: 我们会认真调查所有报告
3. **行动**: 根据调查结果采取适当行动
4. **跟进**: 在适当时候向您更新处理进度

## 后果

违反行为准则可能导致：

- 警告或私下提醒
- 临时或永久禁止参与项目
- 从项目中移除访问权限
- 其他适当的纠正措施

## 申诉

如果您认为您被不公平地处理，可以通过以下渠道申诉：

- 发送邮件到：appeal@your-domain.com
- 联系项目维护者
