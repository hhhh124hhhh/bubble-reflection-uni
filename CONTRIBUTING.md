# 贡献指南

感谢您对心迹泡泡项目的关注！我们欢迎所有形式的贡献，包括但不限于代码提交、问题反馈、功能建议和文档改进。

## 🤝 如何贡献

### 报告问题

- 使用 [GitHub Issues](https://github.com/your-username/bubble-reflection/issues) 报告 bug
- 在提交新问题前，请先搜索确认问题是否已被报告
- 提供详细的问题描述、复现步骤和环境信息
- 包含相关的截图或错误信息

### 功能建议

- 使用 GitHub Issues 提交功能建议
- 详细描述建议的功能和使用场景
- 说明该功能对用户的价值

### 代码贡献

#### 开发环境设置

1. **Fork 项目**
   ```bash
   # 在 GitHub 上 Fork 项目到您的账户
   # 然后克隆到本地
   git clone https://github.com/your-username/bubble-reflection.git
   cd bubble-reflection
   ```

2. **安装依赖**
   ```bash
   npm install
   ```

3. **配置云开发**
   - 创建您的云开发环境
   - 修改 `src/utils/cloudbase.ts` 中的环境 ID
   - 开启必要的登录认证方式

4. **启动开发服务器**
   ```bash
   # H5 开发
   npm run dev:h5
   
   # 或其他平台
   npm run dev:mp-weixin
   ```

#### 代码规范

- 使用 TypeScript 严格模式
- 遵循 Vue 3 Composition API 最佳实践
- 使用有意义的变量和函数命名
- 添加适当的注释和类型注解
- 确保代码通过 ESLint 检查

#### 提交流程

1. **创建分支**
   ```bash
   git checkout -b feature/your-feature-name
   # 或
   git checkout -b fix/your-bug-fix
   ```

2. **开发和测试**
   - 实现您的功能或修复
   - 确保所有测试通过
   - 测试在不同平台的兼容性

3. **提交代码**
   ```bash
   git add .
   git commit -m "feat: 添加新功能描述"
   # 使用规范的提交信息
   ```

4. **推送和创建 PR**
   ```bash
   git push origin feature/your-feature-name
   ```
   - 在 GitHub 上创建 Pull Request
   - 填写详细的 PR 描述
   - 等待代码审查

### 提交信息规范

我们使用 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

- `feat:` 新功能
- `fix:` 修复 bug
- `docs:` 文档更新
- `style:` 代码格式调整（不影响功能）
- `refactor:` 代码重构
- `test:` 测试相关
- `chore:` 构建过程或辅助工具的变动

示例：
```
feat: 添加倒计时任务功能

- 支持自定义倒计时时间
- 添加暂停/继续功能
- 实现进度环可视化显示
```

## 🐛 Bug 报告

提交 bug 报告时，请包含以下信息：

- **问题描述**: 清晰简洁地描述问题
- **复现步骤**: 详细说明如何重现问题
- **期望行为**: 描述您期望发生的情况
- **实际行为**: 描述实际发生的情况
- **环境信息**: 
  - 操作系统和版本
  - 浏览器版本（如果是 H5）
  - 小程序平台和版本
  - Node.js 版本
- **截图/录屏**: 如果可能，提供视觉证据
- **额外信息**: 任何其他有助于理解问题的信息

## 💡 功能建议

提交功能建议时，请包含：

- **问题背景**: 这个功能解决了什么问题？
- **详细描述**: 功能应该如何工作？
- **使用场景**: 用户在什么情况下会使用这个功能？
- **替代方案**: 您考虑过哪些其他解决方案？
- **额外信息**: 任何其他相关信息
