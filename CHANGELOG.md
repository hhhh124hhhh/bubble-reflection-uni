## 版本说明

### 版本号格式
我们使用语义化版本号：`MAJOR.MINOR.PATCH`

- **MAJOR**: 不兼容的 API 修改
- **MINOR**: 向下兼容的功能性新增
- **PATCH**: 向下兼容的问题修正

### 变更类型

- `新增` - 新功能
- `修复` - 问题修复
- `改进` - 现有功能改进
- `文档` - 文档更新
- `样式` - UI/UX 改进
- `重构` - 代码重构
- `性能` - 性能优化
- `安全` - 安全相关更新
- `移除` - 功能移除

### 发布流程

1. **开发版本** - 新功能开发
2. **测试版本** - 功能测试和 bug 修复
3. **候选版本** - 发布前最终测试
4. **正式版本** - 生产环境发布
